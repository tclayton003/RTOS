Dispatch handler - processes a request.

Register using gs_spi_slave_set_rx().

Parameters
[in]deviceSPI device (handle).
[in]rx_bufferPointer to start of rx buffer.
[in]rxnumber of bytes received so far.
[in]new_requesttrue on the first callback of new data, false on receiving additional data during same chip-select. Can be used to bring receiver back in sync with new request.
[in]cswitchSet if called from within an ISR (embedded platforms), otherwise NULL
Returns
total number of bytes to receive before next call back. Return 0 to ignore rest of data - no additional call backs will be done for current SPI transaction.
