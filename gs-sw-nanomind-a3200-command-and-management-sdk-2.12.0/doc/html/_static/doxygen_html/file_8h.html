file.h File Reference

Detailed Description

File based stores.

A file store uses standard file(s) for storing parameters. The store configration contains a path (directory), where each file is slot (= table).

const gs_param_store_driver_t gs_param_store_file_driver_bin

Binary file store (.bin).

Parameters are saved in binary form (big-endian), a long with checksum of table definition and data.

File extension is: bin

const gs_param_store_driver_t gs_param_store_file_driver_cfg

Human readable file store (.cfg).

Parameters are saved in textual form, without any form of checksum/validation.

File extension is: cfg

Example:

# Table board (0)
uid = ""
type = 0
rev = 0
dock_type = ""
csp_addr = 1
csp_rtable = ""
gosh_en = true
gosh_uart = 2
can_en = true
can_brate = 1000000
i2c_en = true
i2c_brate = 400000
kiss_en = false
kiss_brate = 500000
kiss_uart = 4
