1.1. Introduction¶

This manual describes the NanoMind A3200 BSP software package.

It contains software examples of how to build an A3200 application using the NanoMind A3200 BSP.

The following block diagram shows the architecture of the NanoMind A3200 BSP:

Fig. 1.1 NanoMind A3200 BSP Architecture

The User Application is described in User Application.

A number of libraries are included to support development:

  • A3200 (liba3200)
  • Thirdparty (libthirdparty)
  • GomSpace CSP (libgscsp)
  • Embed (libembed)
  • ASF (libasf)
  • Utility (libutil)

More information about the GomSpace software stack can be found in Software concepts and terminology

1.2. Getting started¶

1.2.1. Prerequisites¶

See Programming for installing toolchain, connecting programmer and connecting to GOSH (stdio from A3200).

1.2.2. Unpacking NanoMind A3200 BSP¶

The NanoMind A3200 BSP is shipped as a single archive, which includes all the source code, build scripts, documentation (this document), etc.

Use following steps to extract all files from the archive into a working directory on your Linux system.

  1. Create a workspace folder for the source code, e.g. ~/workspace:

    $ mkdir ~/workspace

  2. Copy the archive to the workspace folder:

    $ cp gs-sw-nanomind-a3200-board-support-package-<version>.tar.gz ~/workspace

  3. Change directory to the workspace folder:

    $ cd ~/workspace

  4. Unpack the archive:

    $ tar xvf gs-sw-nanomind-a3200-board-support-package-<version>.tar.gz

You now have the source code unpacked in: ~/workspace/gs-sw-nanomind-a3200-board-support-package-<version>.

1.2.3. Building NanoMind A3200 BSP¶

Use the following steps to build the NanoMind A3200 BSP.

Building the first time:

  1. Change directory to the NanoMind A3200 BSP folder:

    $ cd ~/workspace/gs-sw-nanomind-a3200-board-support-package-<version>

  2. Bootstrap buildtools:

    $ ./tools/buildtools/gsbuildtools_bootstrap.py

  3. Build using Waf:

    $ ./waf distclean configure build

Subsequent building:

  1. Change directory to the NanoMind A3200 BSP folder:

    $ cd ~/workspace/gs-sw-nanomind-a3200-board-support-package-<version>

  2. Build using Waf:

    $ ./waf

More details about using BuildTools can be found in BuildTools.

1.2.4. Programming NanoMind A3200¶

See Programming for connecting the programmer.

Programming:

  1. Change directory to the NanoMind A3200 BSP folder:

    $ cd ~/workspace/gs-sw-nanomind-a3200-board-support-package-<version>

  2. Program A3200 AVR32 FLASH:

    $ ./waf program

This will write the A3200 image to the AVR32 internal program FLASH and then reset the A3200 to start the new image.
