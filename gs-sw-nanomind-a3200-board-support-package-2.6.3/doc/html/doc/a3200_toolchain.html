2. Programming¶

The NanoMind A3200 can be programmed with either the AVR Dragon JTAG programmer or the Digilent HS3 JTAG programmer.

The following sections describes the details in installing and using both the toolchains for the AVR Dragon programmer and the Digilent HS3 programmer.

2.1. Hardware setup¶

The AVR32 MCU can be programmed with an AVR Dragon JTAG programmer. The AVR Dragon is connected to the AVR32 MCU through a JTAG cable, and an USB cable connects the AVR Dragon to your PC.

Fig. 2.1 Connections beween PC, JTAG, FTDI and the NanoMind A3200

Fig. 2.2 AVR Dragon (left), A3200 (middle) and FTDI/USB (right)

The FTDI cable is made to fit with the 4-pin picoblade connector: GND, VCC, UART RX and UART TX.

See the individual GomSpace product manuals for a more detailed hardware setup.

2.1.1. Powering the NanoMind A3200¶

Warning

Do not power the system using FTDI power and external power supply simultaneously. If an external power supply is used, the VCC line (in the FTDI/USB cable) must be removed from the picoblade connector head.

Warning

Please ensure that any PC/Laptop connected to the USB cable has a properly grounded AC-plug. The external power supply ground and PC/Laptop ground must be the same. Failure to do so will result in Common Mode noise on the GND lines of up to 100+ Volts.

External power: The external power supply must be set to 3.3 volt and a current limiter of 1000 mA. The ground connection is taken from one of the four PCB corners, but could also be made in the USB/Serial cable.

2.2. Using a NanoDock DMC-3¶

The NanoMind A3200 can be fitted on a GomSpace NanoDock DMC-3 with FSI connector. Please refer to the GomSpace NanoDock DMC-3 documentation for information about mounting and powering the NanoMind A3200 module.

Warning

Do not power the system using FTDI power and NanoDock DMC-3 power simultaneously.

2.3. AVR Dragon - JTAG Programmer¶

The A3200 toolchain has been verified on Ubuntu 16.04.5 LTS (http://www.ubuntu.com), with following tools:

  • Python 2.7.12
  • GomSpace AVR32 toolchain 3.4.2_gs1

Install Ubuntu prerequisites:

$ sudo apt install build-essential

The AVR32 toolchain contains cross-compiler for the AVR32 platform and a number of utilities for programing the target platform using a AVR Dragon JTAG programmer.

Install GomSpace AVR32 toolchain:

$ tar xfz gs-avr32-toolchain-3.4.2_gs1.tar.gz
$ cd gs-avr32-toolchain-3.4.2_gs1
$ ./install-avr32.sh

More info about the AVR Dragon can be found on Microchip’s homepage: http://www.microchip.com/developmenttools/ProductDetails/ATAVRDRAGON.

If programming fails with the error message “No USB programmers found.”, it is most likely due to a permission problem.

Verify that the avrtools rule file is installed:

$ ls  /etc/udev/rules.d/
99-avrtools.rules

If the 99-avrtools.rules isn’t present, install the AVR32 toolchain (see Programming) or copy the file from the board support package, when it has been unpacked:

$ sudo cp tools/99-avrtools.rules /etc/udev/rules.d/

Unplug the programmer, wait a few seconds, re-connect the programmer and then re-try the program operation.

2.4. Digilent HS3 - JTAG Programmer¶

The Digilent HS3 JTAG programmer is used to flash the NanoMind A3200 as a replacement of the AVR Dragon JTAG programmer no longer manufactured.

The toolchain for the Digilent HS3 is located in a3200-bsp/tools, and can also be found on the GomSpace GitHub reposetory called avr32-prog. The toolchain uses a pyhton script with dependencies on pyefltools and pyftdi.

To install the dependencies:

$ pip install pyelftools
$ pip install pyftdi

Make sure rules are copied to /etc/udev/rules.d:

$ sudo cp tools/hs3/support/99-ftdi.rules /etc/udev/rules.d/

run:

$ udevadm control --reload-rules
$ udevadm trigger

afterwards, and re-connect the programmer.

To use the programmer run:

$ python3 <PATH-TO>/avr32-prog.py -p digilent_hs3 -R -E -f <PATH-TO>/nanomind-bsp.elf

2.5. FTDI - serial connection (GOSH)¶

When the A3200 NanoMind board boots up, it will will print information on it’s standard out. One of the UARTs on the A3200 is configured as standard in/out.

Once the A3200 board is connected to the PC, using the provided FTDI cable, Linux will create a serial device in /dev/ttyUSB*.

minicom is a small terminal program, which can be used to handle the communication. It can be installed using apt:

$ sudo apt install minicom

Unless otherwise specified for any given GomSpace product, the communication settings are: 500000-8-n-1. This can be specified on the command line, when starting minicom:

$ minicom -b 500000 -8 -D /dev/ttyUSB7

Once minicom is started, it’s configuration can be changed using <ctrl><a> and <z>.

2.6. Visual Studio Code¶

Visual Studio Code (https://code.visualstudio.com/) is a free code editor, with a vast number of features, e.g. IntelliSense (go to definitions). For installation on Linux, please see the online documentation: https://code.visualstudio.com/docs/setup/linux.

Once the tool is installed, navigate to the top folder of your project and start Visual Studio Code by typing:

$ code .

This will launch the graphical interface of Visual Studio Code and you are ready to browse/edit the code:

Fig. 2.3 Visual Studio Code

To learn more about Visual Studio Code and how it works, please see the online documentation https://code.visualstudio.com/docs.
