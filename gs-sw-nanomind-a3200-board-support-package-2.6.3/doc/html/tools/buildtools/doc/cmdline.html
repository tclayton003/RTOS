4.1.6. Using gsbuildtools on Command Line¶

GOMspace buildtools commandline

usage: ./gsbuildtools.py [-h]
                         {where,setup,bootstrap,stylecheck,lib-for-each,status,verify_build,buildinfo,gitinfo,git,revision,jenkinsbuildname,clone,update,cov,dep}
                         ...

4.1.6.1. Sub-commands:¶

4.1.6.1.1. where¶

show where buildtools is located

./gsbuildtools.py where [-h]

4.1.6.1.2. setup¶

create links to Waf/scripts (recursively from current directory)

./gsbuildtools.py setup [-h] [-d] [-n]

4.1.6.1.2.1. Named Arguments¶

-d, --docker

Use docker (default) - deprecated/ignored

Default: False

-n, --no-docker
 

Do NOT use docker

Default: False

4.1.6.1.3. bootstrap¶

run “setup” and mark current directory as project root

./gsbuildtools.py bootstrap [-h] [-d] [-n]

4.1.6.1.3.1. Named Arguments¶

-d, --docker

Use docker (default) - deprecated/ignored

Default: False

-n, --no-docker
 

Do NOT use docker

Default: False

4.1.6.1.4. stylecheck¶

run stylecheck for python and c/c++

./gsbuildtools.py stylecheck [-h] [-s SOURCE] [--python-source PYTHON_SOURCE]
                             [-p PROFILE] [-o OUTPUTFILE] [-e EXCLUDE]
                             [-b BASIC_EXCLUDE]

4.1.6.1.4.1. Named Arguments¶

-s, --source path to sources (glob). Can be used multiple times
--python-source
 path to python sources (glob).
-p, --profile
-o, --outputfile
 
-e, --exclude path to excludes (glob). Can be used multiple times
-b, --basic-exclude
 Can be used multiple times

4.1.6.1.5. lib-for-each¶

run <commands> in each cloned lib

./gsbuildtools.py lib-for-each [-h] [-g] [--ignore-error]
                               commands [commands ...]

4.1.6.1.5.1. Positional Arguments¶

commands

4.1.6.1.5.2. Named Arguments¶

-g, --only-with-gsbuildall
 

run only in dirs with a gsbuildall.py

Default: False

--ignore-error

ignore error(s) - continue

Default: False

4.1.6.1.6. status¶

show git status recursively and verify dependencies.json

./gsbuildtools.py status [-h] [-d] [-m] [-r] [-v]

4.1.6.1.6.1. Named Arguments¶

-d, --develop

Check as if build is done on a feature/develop branch

Default: False

-m, --master

Check as if build is done on a master branch

Default: False

-r, --release

Check as if build is done on a release branch

Default: False

-v, --verbose

verbose output

Default: False

4.1.6.1.7. verify_build¶

Verify build, e.g. sbuildall.py and generated files

./gsbuildtools.py verify_build [-h] [-v]

4.1.6.1.7.1. Named Arguments¶

-v, --verbose

verbose output

Default: False

4.1.6.1.8. buildinfo¶

show build information based on GiT

./gsbuildtools.py buildinfo [-h] [--tag TAG]

4.1.6.1.8.1. Named Arguments¶

--tag parse this tag

4.1.6.1.9. gitinfo¶

show GiT information

./gsbuildtools.py gitinfo [-h] [-r] [-a] [--json] [--develop] [--manifest]
                          [--exclude_path EXCLUDE_PATH [EXCLUDE_PATH ...]]
                          [--manifest-subdir MANIFEST_SUBDIR]

4.1.6.1.9.1. Named Arguments¶

-r, --recursive
 

iterate all GiT modules recursively

Default: False

-a, --all

show all values

Default: False

--json

JSON output (always full listing)

Default: False

--develop

show modules with only develop relation

Default: False

--manifest

write manifest file

Default: False

--exclude_path

specify a list of modules to execlude

Default: []

--manifest-subdir
 put manifest files in a sub-directory

4.1.6.1.10. git¶

various git commands and wrappers

./gsbuildtools.py git [-h] [--move-latest-tag]

4.1.6.1.10.1. Named Arguments¶

--move-latest-tag
 

Move (or create) a -latest tag based on the current version tag

Default: False

4.1.6.1.11. revision¶

GiT revision

./gsbuildtools.py revision [-h] [-f FILE]

4.1.6.1.11.1. Named Arguments¶

-f, --file write to file

4.1.6.1.12. jenkinsbuildname¶

legacy command - use revision

./gsbuildtools.py jenkinsbuildname [-h] [-f FILE]

4.1.6.1.12.1. Named Arguments¶

-f, --file

Default: “artifacts/version.txt”

4.1.6.1.13. clone¶

clone all project dependencies (dependencies.json)

./gsbuildtools.py clone [-h] [-f] [-b BRANCH | -H | -M | -S | --version | -l]
                        [-v] [-r] [-p]
                        [libs [libs ...]]

4.1.6.1.13.1. Positional Arguments¶

libs

specify a list of libs to clone. Default is all

Default: []

4.1.6.1.13.2. Named Arguments¶

-f, --force

deletes path if it exists

Default: False

-b, --branch

branch, default is “develop”

Default: “develop”

-H, --head

when cloning based on tag, resolved the containing branches and pick the best one based on GS branch model

Default: False

-M, --checkout-major
 

checkout based on major version - requires min_version field for each dependency in dependencies.json

Default: False

-S, --sha

checkout based on sha - requires a sha field for each dependency in dependencies.json

Default: False

--version

Checkout the sha if specified or the specific version specified in min_version

Default: False

-l, --latest

Checkout the sha if specified or use the min_version as a major spec and if found checkout -latest version of tag

Default: False

-v, --verbose

verbose output

Default: False

-r, --recursive
 

If the dependency in dependencies.json has recursive: true buildtools will go into the cloned folder and clone its dependencies

Default: False

-p, --parallel

clone the dependencies in separate threads. Use with caution

Default: False

4.1.6.1.14. update¶

updates dependencies.json

./gsbuildtools.py update [-h] [-f]

4.1.6.1.14.1. Named Arguments¶

-f, --force

updates even when repo is not clean

Default: False

4.1.6.1.15. cov¶

generate gocv output in human readable html. (requires GS gcovr)

./gsbuildtools.py cov [-h] [-b] [-t] [-c]

4.1.6.1.15.1. Named Arguments¶

-b, --no-browser
 

Do not try to open the output in a browser

Default: False

-t, --test

generate xml file and verify that all files in src are included in the xml output

Default: False

-c, --clean

clean up unwated gcda and gcno files to get only the files needed by the current project

Default: False

4.1.6.1.16. dep¶

show dependencies

./gsbuildtools.py dep [-h] [--csv] [-m MANDATORY [MANDATORY ...]]

4.1.6.1.16.1. Named Arguments¶

--csv

CSV list

Default: False

-m, --mandatory
 Print warning if mandatory dependency is missing
