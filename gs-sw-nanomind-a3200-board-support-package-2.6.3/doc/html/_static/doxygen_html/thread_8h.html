The thread API wraps POSIX pthread and FreeRTOS task.

Enumeration Type Documentation

enum gs_thread_priority_t

Thread priorities.

These values are mapped to platform specific values.

Enumerator
GS_THREAD_PRIORITY_IDLE 

Idle (lowest) priority.

Typical use: Not much - runs when nothing else runs. FreeRTOS: Idle thread.

GS_THREAD_PRIORITY_LOW 

Low priority.

Typical use: Service applications, e.g. servicing requests from the outside. GOMspace: housekeeping, GOSH.

GS_THREAD_PRIORITY_NORMAL 

Normal priority.

Typical use: Control - the primary application(s).

GS_THREAD_PRIORITY_HIGH 

High priority.

Typical use: Drivers off loading data from hardware to software buffers. GOMspace: csp_route_task.

GS_THREAD_PRIORITY_CRITICAL 

High priority.

Typical use: Very time critical threads. No long, time consuming processing. FreeRTOS: Timer thread.

Create thread (or task on some platforms).

pthread/Posix supports exit value and join, FreeRTOS supports neither (perhaps in the future), so API is designed after Posix.

FreeRTOS: a thread must always terminate with a call to gs_thread_exit(). linux: a thread is by default created detached, unless GS_THREAD_CREATE_JOINABLE is specified.

Parameters
[in]namename of thread. Ignored on Linux.
[in]funcfunction for thread to execute.
[in]parameterparameter parsed to the thread function.
[in]stack_sizenumber of bytes to allocate for stack - not used/supported on all platforms. Ignored on Linux.
[in]prioritythread priority. Ignored on Linux.
[in]flagsflags to control creation, see GS_THREAD_CREATE_JOINABLE.
[out]handlehandle to the created thread, use NULL if not wanted.
Returns
GS_OK on success
other error code on failure

Create thread (or task on some platforms) with user supplied buffer for stack.

pthread/Posix supports exit value and join, FreeRTOS supports neither (perhaps in the future), so API is designed after Posix.

FreeRTOS: a thread must always terminate with a call to gs_thread_exit(). FreeRTOS v9.0 must be compiled with configSUPPORT_STATIC_ALLOCTION set to 1 - otherwise warning log is printed and user supplied stack buffer is discarded linux: a thread is by default created detached, unless GS_THREAD_CREATE_JOINABLE is specified. stack_buf is ignored.

Parameters
[in]namename of thread. Ignored on Linux.
[in]funcfunction for thread to execute.
[in]parameterparameter parsed to the thread function.
[in]stack_sizesize of the user supplied stack buffer - not used/supported on all platforms. Ignored on Linux.
[in]stack_bufUser supplied stack buffer - not used/supported on all platforms. Ignored on Linux.
[in]prioritythread priority. Ignored on Linux.
[in]flagsflags to control creation, see GS_THREAD_CREATE_JOINABLE.
[out]handlehandle to the created thread, use NULL if not wanted.
Returns
GS_OK on success
other error code on failure
